¿Qué son los patrones de diseño?

Son soluciones probadas y documentadas para problemas recurrentes en el desarrollo de software. Ayudan a crear sistemas robustos, mantenibles y escalables.


1. Patrones Creacionales

Se encargan de los mecanismos de creación de objetos.

Patrón Descripción
Singleton Garantiza una única instancia de una clase.
Factory Method Proporciona una interfaz para crear objetos en una superclase.
Builder Permite construir objetos complejos paso a paso.

2. Patrones Estructurales

Explican cómo montar objetos y clases en estructuras más grandes.

  • Adapter: Permite que objetos con interfaces incompatibles colaboren.
  • Facade: Proporciona una interfaz simplificada a una biblioteca o sistema complejo.
  • Proxy: Proporciona un sustituto o marcador de posición para otro objeto.

3. Patrones de Comportamiento

Se encargan de la comunicación y asignación de responsabilidades entre objetos.

Observer
Define un mecanismo de suscripción para notificar a varios objetos sobre eventos.
Strategy
Permite definir una familia de algoritmos y hacerlos intercambiables.
State
Permite que un objeto altere su comportamiento cuando su estado interno cambia.
